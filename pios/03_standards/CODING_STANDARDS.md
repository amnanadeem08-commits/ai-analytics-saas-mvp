# Coding Standards

1. Prefer small, focused services and pure helpers over fat UI modules
2. Keep Streamlit pages thin — move logic to `frontend/services` or backend services
3. Do not invent APIs or features not evidenced in the repo
4. Match existing naming and module layout before introducing new packages
5. Preserve backward-compatible defaults when adding optional flags
6. Secrets and keys via environment variables only
