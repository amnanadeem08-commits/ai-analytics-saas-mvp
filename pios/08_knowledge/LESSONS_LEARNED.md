# Lessons Learned

From sprint history and v1.0 release:

- Keep services behind interfaces (repos, storage, queue) for swap-outs
- In-memory MVP stores ship faster but must be documented as limitations
- Release gates (tests + E2E + checklists) beat feature count for production trust
- Registry/plugin architecture scales domain behavior better than hardcoded branches
- Evidence-based documentation (“Not verified”) prevents handbook lies

Update this file at sprint close when a lesson is new and durable.
